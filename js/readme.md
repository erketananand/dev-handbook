### Differences between **dependencies** and **devDependencies** found in a `package.json` file:

  * `dependencies` are packages required for the application to run in production.
  * `dependencies` usually include core libraries like **UI libraries**(`react`, `react-dom`), **State management**(`redux`, `zustand`), **HTTP clients**(`axios`), **Backend frameworks**(`express`, `fastify`), **Database clients**(`mongoose`, `pg`) & **Utility libraries**(`lodash`, `date-fns`).
  * `devDependencies` are tools used only during development such as testing, linting, or building.
  * `devDependencies` include **Build Tools**(`webpack`, `vite`, `parcel`, `rollup`), **Transpilers / Compilers**(`@babel/core`, `@babel/preset-react`, `typescript`), **Linters**(`eslint`, `prettier`), **Testing Libraries**(`jest`, `mocha`, `vitest`) & **Development Utilities**(`nodemon`, `concurrently`, `husky`).*
