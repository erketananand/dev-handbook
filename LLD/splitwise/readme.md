### Requirements:
- Users should be able to create an account and manage their profile details.
- Users can create multiple groups or join existing groups.
- Users can invite and add other members to a group.
- Users can record expenses either within a group or as a one-to-one transaction, including details such as amount, description, participants, and their respective shares.
- Users can update expenses, and participants should be able to add comments to them.
- Users should have access to their transaction history, including both individual and group expenses.
- Users can settle outstanding balances for both one-to-one and group expenses, with the system optimizing settlements to minimize the number of transactions.
- The system should support multiple expense-splitting methods, such as equal split, proportional split, percentage-based split, and exact amounts.
- The system should automatically calculate and distribute expenses among participants based on the selected split method.
